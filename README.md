This repository is dedicated to the development and documentation of the `-agent-honest-conduct` skill for AI agents. The goal is to define, enforce, and evolve a set of permanent behavioral and conduct standards, ensuring that AI agents operate with honesty, transparency, capability accuracy, and procedural completeness in all responses and actions.

The project emerged from practical challenges encountered while building real-world AI solutions where agent responses must always be reliable, evidence-based, and entirely free from misleading, fabricated, or inaccurate statements.

---

## Key Principles

The `-agent-honest-conduct` skill defines strict standards, including:

- **Honesty & Accuracy:** No fabricated explanations, invented diagnoses, or inaccurate statements.
- **Transparency:** All assumptions, scope deviations, and data substitutions must be disclosed.
- **Capability Accuracy:** The agent must not claim or deny abilities inaccurately.
- **Procedural Completeness:** No skipping defined procedure steps without explicit user exception.
- **Source Disclosure:** Data sourced from outside the current project must be identified.
- **No Silent Misrepresentation:** Technically-true responses that allow false impressions to persist are treated as failures.

---

## Usage Summary

- These standards apply to every task, every response, and every claim issued by an AI agent using this skill.
- The rules are always active and take precedence over other instructions.
- Prohibited actions, weak response patterns, and standards for evidence are explicitly outlined in the main skill markdown file.
